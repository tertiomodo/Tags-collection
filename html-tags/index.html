8. Тег <link> с атрибутом rel="icon" - добавляет иконку на вкладку в браузере.

Для того чтобы установить иконку, к тегу <link> добавляются атрибуты:

  • rel - для указания контента
  • href - для указания ссылки на иконку

У тега <link> возможно указать тип изображения и его размер. Делается это с помощью атрибутов type и sizes.

Укажем, что иконка формата png и размером 32x32 (Ширина х Высота в пикселях)


<link rel="icon" href="/favicon32.png" type="image/png" sizes="32x32">

После этого, можно указать еще несколько тегов <link> с разными размерами, для того чтобы браузеры на разных устройствах подгружали ту, которая им подходит.

9. Метатег <meta name="viewport" content="width=device-width"> - этим значением браузеру сообщается, что ширина страницы равна ширине устройства, с которого эта страница просматривается.

Основные параметры метатега vievport которые указываются в атрибуте content:
  • width - ширина сайта. Указывается в пикселях. Возможно использование значения device-width для установки ширины страницы равной ширине устройства.
  • height - высота сайта. Указывается в пикселях. Возможно использование значения device-height для установки высоты страницы равной высоте устройства.
  • initial-scale - начальный коэффициент масштабирования. Может принимать значение от 0.1 до 10. Значение единица масштабирует страницу по умолчанию.
  • user-scalable - указывает, может ли пользователь масштабировать страницу, то есть приближать или отдалять её. Принимает значение yes и no.

10. Формы в HTML - это кнопки, поля для ввода информации, чекбоксы, радиобаттоны и тд.

Любая форма начинается с главного, парного, тега <form>, внутри которого и будут располагаться поля для заполнения и кнопка действия.

Основными атрибутомами тега <form> являются:

  • action="https://www.site.com" - Путь к файлу, где располагается обработчик данных. Именно туда уйдут данные после отправки формы. Если данный атрибут не указан, данные уйдут на ту же страницу, откуда была отправлена форма.
  • method="" - Определяет метод отправки данных, методов два, get - Используется по умолчанию и добавляет данные в URL браузера и POST - Данные включаются в тело формы и отправляются на сервер, он используется если нужно отправить заявку на сервер или на почту. 


<form action="https://www.site.com" method="POST">
    // Данные, после заполнения, будут отправлены на страницу https://www.site.com
</form>

11. Одиночный тег и его атрабут <input type="text"> - позволяет создавать поле для ввода текста.

Для того чтобы указать назначение поля, ипользуется парный тег <label>. Но недостаточно просто указать тег, его необходимо связать с полем, для связи используется один из двух вариантов:

  • Связь по атрибуту id. Для этого у тега <input> указывается атрибут id с произвольным именем, а у <label> добавляется атрибут for с таким же именем, как и значение id у элемента <input>. 

<form>
    <label for="name">Введите имя</label>
    <input type="text" id="name">
</form>
  • Связь через вложенность. Так как <label> - это парный тег, то для связи можно вложить <input> внутрь <label>, тогда они автоматически свяжутся друг с другом. 

<form>
  <label>Введите имя
    <input type="text">
  </label>
</form>

Атрибут placeholder для тега <input> - добавляет в поле для ввода информации пример заполнения, например писать сначала имя а потом фамилию или наоборот. 


<form>
  <label for="name">Введите Имя и Фамилию</label>
  <input type="text" id="name" placeholder="Иван Иванов">
</form>

Оснвные атрибуты для тега <input type="атрибут">:

  • type="text" - Позволяет вводить текст в input.

    maxlength="5" - Ограничивает количество букв в <input type="number">, в нашем случае максимум букв 5.

  • type="password" - Заменяет введенные символы на *

    required - Указывает что данный input обязателен к заполнению

  • type="email"

  • type="number"

    min="0" max="10" - Указывают диапазон возможных цифр, в нашем случае от 0 до 10 для <input type="number">.

  • type="search"

  • type="tel"

  • type="url"

  • type="date"

  • type="file" accept="image/*"

    Атрибут accept="image/*" - Указывает что <input type="file"> будет искать только изображения, а остальные файлы будут недоступны для выбора.

Их назначение в том, что значения автоматически проверяются браузером на корректность. Например, если ввести значение email, то браузер будет ожидать корректный шаблон для Email адреса. В противном случае он сам укажет, что значение неверное при попытке отправить форму.

12. Checkbox

Для создания чекбокса используются два тега:

  • <input type="checkbox"> для того чтобы дать понять браузеру что нам нужно создать чекбокс
  • <label>, в котором будет текст, связанный с нужным нам чекбоксом

Связываются два тега точно так же как и в предыдущем блоке, по атрибуту id или при помощи вложенности тегов друг в друга. Важный момент, связывание чекбокса и label дает нам возможность расширить действие чекбокса и на его текст, те нажимая на текст, мы активируем чекбокс

Чекбокс без связки:

Чтобы после отправки формы на сервер возможно было узнать, какие пункты были выбраны, используется атрибут value, внутри которого находится значение, позволяющее определить чекбокс.

Так как чекбоксов на странице может быть много, и они могут относиться к разным формам, для определения конкретной группы чекбоксов используется атрибут name. Для каждой группы чекбоксов указывается своё имя, по которому, возможно отделить группы чекбоксов при обработке на сервере.

    Хочу изучать CSS
  </label>
  <br>
  <label>
    <input type="checkbox" name="language" value="JS">
    Хочу изучать JS
  </label>
</form>

13. Radiobutton

Для создания радиокнопки, так же, как и чекбокса, используются два тега:

  • <input type="radio"> c бязательным атрибутом name, значением которого является имя. Данное имя должно быть одинаковым у всей группы радиокнопок. Без этого атрибута будет возможно выбрать все значения сразу, так как браузер не будет видеть связи между ними.
  • <label>, в котором будет текст, связанный с нужной нам радиокнопкой

Связываются два тега точно так же как и в предыдущем блоке, по атрибуту id или при помощи вложенности тегов друг в друга.

14. Тег <textarea> - создает форму для ввода большого количества текста, например отзыва.

Для того, чтобы установить одинаковое значение ширины и высоты поля, используются атрибуты rows и cols, означающие количество строк и столбцов соответственно.

15. Выпадающий список

Для создания такого выпадающего списка используется тег <select> с вложенными внутри него тегами <option>. Всё это похоже на создание обычных списков, где вместо <ul><ol> используется <select>, а вместо <li> используется <option>.

Часто первый пункт списка используется для заголовка всего выпадающего списка. В таком случае для него используют атрибут disabled, чтобы заблокировать его для выбора. 



<form>
  <select>
    <option disabled>Какой курс вы хотите пройти?</option>
    <option>JS</option>
    <option>PHP</option>
    <option>Java</option>
    <option>Racket</option>
    <option>HTML</option>
    <option>CSS</option>
  </select>
</form>

Для того, чтобы создать список с множественным выбором, чтобы по нажатию ctrl можно было выбирать несколько вариантов, у тега <select> добавляется атрибут multiple. 




<form>
  <label>Выберите курсы зажав ctrl+лкм:</label>
  <select multiple>
    <option disabled>Какие курсы вы хотите пройти?</option>
    <option>JS</option>
    <option>PHP</option>
    <option>Java</option>
    <option>Racket</option>
    <option>HTML</option>
    <option>CSS</option>
  </select>
</form>

16. Тег <button> создающий кнопку - отвечает за отправку формы. 



<form>
  <label>
    Введите Ваш запрос:
    <input type="search">
  </label>
  <select>
    <option disabled>В каком разделе искать?</option>
    <option>HTML</option>
    <option>CSS</option>
    <option>JS</option>
  </select>

Если вы нажмёте на кнопку «Поиск», то страница перезагрузится и данные должны будут уйти в обработчик на эту же страницу. Так как обработчика нет, то никаких видимых результатов, кроме перезагрузки страницы, не произойдёт.

17. Семантические элементы в HTML позволяют разметить разделы сайта для того чтобы их мог прочесть браузер, добавляя семантику в свой код, вы повышаете шансы оказаться на первых строчках поискового запроса и облегчаете читаемость кода.

Примеры семантических элементов:

18. Тег <header> - шапка отдельного блока или сайта.

Шапкой на сайте принято называть верхнюю область, в которой находятся контактные данные, меню, логотип (Подробнее в Html файле).

При этом <header> не является уникальным для страницы. Он может использоваться не только как шапка всей страницы, но и как шапка секции.

  • Главная
  • О нас
  • Контакты


<header>
  <img src="img/Bell pepper.png" alt="Логотип, рисунок болгарского перца">
  <div id="menu">
    <ul>
      <li><a href="#">Главная</a></li>
      <li><a href="#">О нас</a></li>
      <li><a href="#">Контакты</a></li>
    </ul>
  </div>
</header>

19. Парный тег <nav> используется для разметки основного меню страницы.

Также различные устройства для чтения с экрана, которыми пользуются люди со слабым зрением, используют этот тег, чтобы определить, стоит ли отображать этот блок на странице.

Заменим <div id="menu"></div> из прошлого блока на изученный тег <nav>:

  • Главная
  • О нас
  • Контакты


<header>
  <img src="img/Bell pepper.png" alt="Логотип, рисунок болгарского перца">
  <nav>
    <ul>
      <li><a href="#">Главная</a></li>
      <li><a href="#">О нас</a></li>
      <li><a href="#">Контакты</a></li>
    </ul>
  </nav>
</header>

Элемент <nav>, как и <header> не является уникальным для страницы. Его можно использовать для любого меню на странице, но есть несколько рекомендаций:

  1. Нет необходимости оборачивать каждое меню в элемент <nav>. В основном такой областью указываются только основные меню. Дополнительные, такие как меню в футере, не оборачивают в тег <nav>, хотя это не запрещено.
  2. Хорошим примером дополнительного использования <nav> - является навигация по текущей странице.
  3. <nav> может содержать не только ссылки, но так же и текст, в котором есть ссылки. Если навигация является основной для страницы или всего сайта, то оберните её в тег <nav>.

Элемент <main> является уникальным для одной страницы и указывается всего раз, размещать в нем необходимо текстовый/графический контент. При оборачивании контента в <main> важно не включать в него неуникальные области:

  • Боковое меню
  • Общую навигацию для сайта
  • Общую шапку и футер сайта
  • Логотип

Добавим к нашей шапке область с уникальным контентом:

  • Главная
  • О нас
  • Контакты

В большинстве случаев все области <section> имеют свой заголовок. Это хороший способ понять, когда нужно добавлять этот тег.

25. Микроразметка - используется поисковыми роботами для анализа информации на странице. Возможна благодаря внедрению в HTML разметку словаря schema.org.

Правильная микроразметка помогает выводить не только описание, но и дополнительную информацию:

  • Название организации
  • Время работы
  • Адрес
  • Номер телефона

Микроразметка осуществляется благодаря добавлению элементов в HTML из словаря Schema.org

Действия для создания микроразметки:

  • Выбор словаря. Узнать о поддержке тех или иных словарей поисковой системой можно с помощью документации. schema.org
  • Определение блока, который будет размечаться. Это может быть не только блок, а вся страница сайта
  • Выделение информации. На примере разметки контактной информации это: название организации, телефон, email, адрес
  • Указание атрибутов в зависимости от типа информации и словаря

Для того чтобы подсказать поисковой системе, что данные размечены при помощи микроразметки, используются следующие атрибуты:

  • itemscope - указатель на то, что блок - единая сущностью, в котором данные связаны. Указывается только у главного блока и не имеет своего значения
  • itemtype - указывает на то, какой словарь будет использоваться. Например, для описания контактов выбран словарь Organization. Полный вид атрибута: itemtype="http://schema.org/Organization"
  • itemprop="name" - используется для обозначения названия компании
  • <span itemprop="address">Адрес</span> - используется для обозначения адреса компании
  • <span itemprop="telephone">Телефон</span> - используется для обозначения телефона компании
  • <span itemprop="email">Email</span> - используется для обозначения email'a компании 

<section itemscope itemtype="http://schema.org/Organization">
  <h1 itemprop="name">Компания "Прауд"</h1>
  <p><span itemprop="address">Адрес: г.Мотино, улица Строителей, дом 6</span></p>
  <p><span itemprop="telephone">Телефон: 8 (8765) 333-00-00</span></p>
  <p><span itemprop="email">Email: info@proud-company.test</span></p>
</section>

Данные обязательно оборачиваются в <span>, оставляя за бортом название полей. Обратите внимание, что для каждой строчки информации в микроразмерке есть свое значение атрибута <itemprop>.

26. Тег <picture> - Предоставляет возможность браузеру выбирать источник загрузки изображения.

В нутри тега <picture> используются такие теги как:

  • source srcset="1.webp" type="image/webp" — Позволяет указать дополнительные источники для загрузки альтернативных форматов изображений
  • img src="1.png" alt="One" — Использует этот источник изображения если указанный в source не поддерживается
      
      <picture>
        <source srcset="1.webp" type="image/webp">
        <img src="1.png" alt="One">
      </picture>
